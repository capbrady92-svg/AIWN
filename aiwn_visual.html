AIWN — Adaptive Indexed Weight Networks

A new class of neural network layer.
Each weight knows which input is coming.

Standard layers use one fixed weight per connection — the same value for every possible input, always. AIWN gives each weight K versions and uses the input itself to instantly select which two to blend. The result: same parameters stored, fraction of them read, same or better accuracy.

01 · The Problem
GPUs spend most of their time reading weights they don't need.
When a neural network answers a question, the GPU must first load the weight matrix from VRAM into registers before it can multiply anything. That memory read — not the arithmetic — is the real cost. And standard networks read 100% of the weight matrix for every single input, regardless of what the input actually is.
The memory hierarchy problem
bottleneck
Each level is orders of magnitude faster but smaller. Reading from VRAM is the bottleneck — 100× slower than arithmetic.
Standard MLP — what actually happens
100% reads
Every forward pass reads the entire weight matrix — even though only a fraction of it is relevant to this particular input.
02 · The Insight
Each weight has K versions. The input value picks two.
Instead of one fixed weight w[i][j], each parameter becomes a small table w[i][j][k] with K entries. The input value is used to instantly compute which two entries to blend — no learned gate, no routing network, just one multiply and one floor operation. Think of it as a deck of cards: instead of always playing the same card, the input reaches into the deck and pulls out the two cards nearest to where it lands.
MLP — w[i][j]
one card, always
output = x · w
w is one scalar. No knowledge of x. Identical for all inputs.
AIWN — w[i][j][k]
deck of K cards
k = floor(x·K)  ·  frac = x·K − k
output = x · lerp(wₖ, wₖ₊₁, frac)
One multiply + one floor. Index is essentially free to compute.
03 · Why It's Fast
Index computation is free. Memory savings are enormous.
The index k = floor(x·K) costs one multiply and one floor — effectively zero GPU cycles. But the payoff is that instead of reading d×d weights from VRAM, you read 2×(d/√K)² — a fraction. At K=32 that's ~6% of the weight matrix per forward pass. The other 94% never leave VRAM. They're stored, but never touched.

Why memory reads cost so much
VRAM bandwidth is ~1–3 TB/s on modern GPUs. Sounds fast — but a 7B parameter model at FP16 is 14GB. Reading it fully takes ~5–14ms just in memory transfer, before a single multiply happens. Tensor cores sit idle waiting for data. AIWN keeps them fed.
Why the index is essentially free
k = floor(x · K) runs in a single GPU instruction. No neural network, no softmax, no load balancing loss needed. The input value directly encodes which bucket it belongs to — deterministic, differentiable via the lerp, and zero overhead.
04 · Why Accuracy Holds — and the Network Can Be Smaller
K local fits always beat one global fit. Smaller networks become viable.
A standard layer uses one W to approximate the mapping across all inputs simultaneously — a forced global compromise. AIWN decomposes this: each bucket only needs to approximate its narrow slice of input space. This is classical piecewise approximation theory — proven for over a century. More pieces = better fit, always. And because each bucket covers less territory, it needs less capacity — enabling smaller d per slice.
